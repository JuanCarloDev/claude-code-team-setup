# Notify Team

## Overview

After any significant dev session, communicate what changed to the team. A short message with links and a clear ask to review. No one should be surprised by live changes.

## When to Use

- Major feature shipped (new sections, templates, product pages)
- Theme pushed to live / published
- Homepage layout changed
- Templates assigned or product pages updated
- Any change visible to customers

## Template

```
Subject: 🐧 [Area] Update — Please Review

Hey team,

Just shipped: [1-line summary of what changed].

**What's new:**
- [Change 1]
- [Change 2]
- [Change 3]

**Review links:**
🏠 Homepage → https://penguin-method.com
[+ relevant product/page URLs]

Let me know if anything looks off — backup ready to revert instantly.

[Name]
```

## Rules

- Always in **English**
- Keep subject line under 60 chars
- Bullet points only — no paragraphs
- Always include "backup ready to revert" when theme was published
- List only links relevant to what actually changed
